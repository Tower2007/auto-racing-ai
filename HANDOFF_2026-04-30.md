### 2-2. 根本原因(2 種類のエラー)

#### エラー A: 08:00 で観測された `np.log1p NoneType`

```
File "daily_predict.py", line 107, in _engineer_odds
    df["log_win_odds"] = np.log1p(df["win_odds"])
TypeError: loop of ufunc does not support argument 0 of type NoneType
```

- 直前の修正 `91085a6` で `parse_odds_summary` (`src/parser.py:241`) は
  `tnsOddsList`/`fnsOddsList` が list(空)の早朝でもクラッシュしないようになった
- しかし `playerList` 自体は埋まっているため、`win_odds=None` の行を返す
- 下流 `daily_predict.py:_engineer_odds` (line 105–114) はそれを想定しておらず
  `np.log1p(None)` で落ちる
- `build_features_for_race` (line 137–140) は `odds.empty` 判定だけで通している:
  ```python
  odds = pd.DataFrame(odds_rows) if odds_rows else pd.DataFrame()
  if not odds.empty:
      ...
      odds = _engineer_odds(odds)   # ← ここで落ちる
  ```

#### エラー B: 06:16–06:20 で観測された `'list' object has no attribute 'get'`

- `predict_race` (`daily_predict.py:239`) の早期 return ガードを通った後で発生
- 候補: `parse_program_entries` か `parse_program_stats` のどこかで
  list が想定される dict にぶつかる(早朝はプログラム本体すら未公開のケース)
- `predict_race` 内の `try/except Exception as e` で握り潰され、
  `logging.error("predict_race(...) failed: %s", e)` の形で出力されている
- 08:00 ではこの形のエラーは消えている → プログラム JSON は 8 時時点で公開済、
  オッズだけ未公開、というのが本日の状態

### 2-3. 修正方針(別 PC で着手)

**最低限の修正:** `_engineer_odds` の手前で win_odds が全 None なら odds 列をスキップする。

`daily_predict.py:137-141` 付近:

```python
odds = pd.DataFrame(odds_rows) if odds_rows else pd.DataFrame()
if not odds.empty:
    # オッズ未公開: win_odds が全 None なら特徴量化をスキップ(NaN のまま merge)
    if odds["win_odds"].isna().all():
        odds = pd.DataFrame()  # 空にして以降の merge / engineer をスキップ
    else:
        odds["race_date"] = pd.to_datetime(odds["race_date"])
        odds = _engineer_odds(odds)
```

ただし戦略は **EV ベース**(`pred_calib * (place_odds_min+max)/2 >= 1.50`)で、
オッズなしでは EV 計算自体ができないので、上流の `predict_race` (line 252–255) で
「オッズ未公開なら早期 return」する方が筋が通る:

```python
odds_resp = client.get_odds(place_code, race_date, race_no)
odds_body = odds_resp.get("body", {})
if isinstance(odds_body, list):
    return pd.DataFrame()
# 追加: tnsOddsList が dict でない / 空 dict なら未公開扱いで早期 return
tns = odds_body.get("tnsOddsList") if isinstance(odds_body, dict) else None
if not isinstance(tns, dict) or not tns:
    return pd.DataFrame()
```

これで 08:00 のような早朝に「候補なし」メールが鳴ることもなくなる
(オッズ公開後に再走させれば普通に動く)。

エラー B 側は `traceback.format_exc()` がログに出ているはずなので、別 PC で
`grep -A 30 "predict_race(6, 2026-04-30, 1) failed" data/daily_predict.log`
で発生位置を特定してから個別修正。
